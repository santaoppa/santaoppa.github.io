## 개념

### MCP (Model Context Protocol)
- AI 모델이 도구 및 데이터 소스와 쉽게 상호작용하도록 설계된 개방형 프로토콜
- 서비스마다 다른 연동 방식을 사용할 필요 없이, AI가 일관된 방식으로 도구를 사용할 수 있도록 표준화함

### API (Application Programming Interface)
- 소프트웨어와 소프트웨어가 서로 데이터를 주고받거나 기능을 사용할 수 있도록 제공하는 인터페이스
- 서비스마다 호출 방식, 인증, 요청/응답 형식, 오류 처리 등이 달라 개발자가 각각 구현해야 함
<br>
<br>

## 예시 : 회의실 예약
회의실 예약 기능을 개발한다고 가정해보자.

### API 방식
개발자는 회의실 조회 API와 예약 API를 직접 호출하고, 응답을 처리하는 로직을 구현해야 함
```java
roomApi.getRooms();
roomApi.reserve(roomId, date, time);
```


- 회의실 조회 API와 호출
- 예약 API와 호출
- 응답/예외 처리
<br>
<br>

### MCP 방식
사용자는 AI에게 자연어로 요청
> 내일 오후 2시에 6명 회의실 예약해줘   
   
그러면 AI는 Metting Room MCP Server를 통해
```text
사용자
   │
   ▼
   AI
   │
   ▼
Meeting Room MCP Server
   │
   ├── 회의실 조회
   ├── 예약 가능 여부 확인
   └── 회의실 예약
``` 
을 순차적으로 수행한 뒤 결과를 사용자에게 전달함
<br>
<br>


## 💡 핵심 차이

| API | MCP |
|------|------|
| 개발자가 API를 직접 호출한다. | AI가 MCP를 통해 적절한 도구를 호출한다. |
| 서비스마다 다른 호출 방식과 인증을 구현해야 한다. | AI는 표준화된 방식으로 다양한 도구를 사용할 수 있다. |
| 응답을 파싱하고 비즈니스 로직을 개발자가 구현한다. | AI가 결과를 해석하고 사용자에게 자연어로 전달한다. |

<br>
<br>

> 💡한 줄 요약  
>  API는 개발자가 호출, MCP는 AI가 호출한다.
