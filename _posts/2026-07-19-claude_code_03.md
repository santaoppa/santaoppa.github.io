# 출력 스타일 (/output-style)

- Claude Code의 시스템 프롬프트를 수정하여 응답 방식을 변경하는 기능
- 기본 제공 스타일뿐만 아니라 사용자 정의 스타일도 만들 수 있음

<br>
<br>

# 내장 출력 스타일

| 스타일 | 설명 |
|--------|------|
| **Default** | 간결하고 직접적인 응답 |
| **Explanatory** | 결정 근거와 대안을 포함한 상세한 설명 |
| **Learning** | 학습자를 위한 교육적인 설명과 용어 해설 제공 |
| **Custom** | 사용자가 직접 정의한 출력 스타일 |

<br>
<br>

# 사용자 정의 출력 스타일
- 사용자 정의 출력 스타일은 Markdown 파일로 작성
- Claude Code는 해당 파일의 내용을 시스템 프롬프트에 추가하여 응답 방식을 변경함


```md
# Style Name

이 스타일에 대한 간단한 설명.

## Instructions

Claude Code에 추가할 지침

- 지침 1
- 지침 2
- 지침 3

## Example Behavior

이 스타일을 사용할 때 예상되는 동작 예시
```
<br>

> 💡 **중요** : keep-coding-instructions: true로 설정하여 클로드 코드의 코딩 능력(테스트, 에러 검증, 파일 수정 등)을 유지

<br>


## 예제


```md
---
name: Beginner Friendly
description: 초보자를 위한 자세한 설명 스타일
keep-coding-instructions: true
---

# Beginner Friendly Style

당신은 프로그래밍을 배우는 초보자를 돕는 친절한 선생님입니다.

## 주요 행동 방식

- 코드를 한 줄씩 자세히 설명하고, 왜 이렇게 하는지 이유를 함께 알려주세요
- 예제 코드에는 한글 주석을 충분히 달아주세요
- 어려운 프로그래밍 용어는 쉽게 풀어서 설명하세요
- 비유를 많이 사용하여 이해하기 쉽게 설명하세요
- HTML 구조를 먼저 설명한 후 JavaScript 로직을 설명하세요
```

<br>

## 저장 위치
출력 스타일은 사용자 전체 또는 프로젝트별로 저장할 수 있음

| 범위 | 저장 위치 |
|------|-----------|
| 사용자(User) | `~/.claude/output-styles/` |
| 프로젝트(Project) | `./.claude/output-styles/` |

<br>
<br>

# 언제 사용하면 좋을까?
- 답변을 짧게 또는 자세하게 고정하고 싶은 경우
- 코드 리뷰 스타일을 통일하고 싶은 경우
- 팀의 문서 작성 규칙을 적용하고 싶은 경우
