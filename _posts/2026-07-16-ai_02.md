# CLAUDE.md

`CLAUDE.md`는 프로젝트 루트에 위치하는 Markdown 파일로, **Claude Code가 세션 시작 시 자동으로 컨텍스트에 포함하여 읽는 파일**이다.

프로젝트의 개발 규칙, 워크플로우, 아키텍처 등 Claude가 항상 참고해야 하는 내용을 정의할 수 있다.
<br>
<br>


# CLAUDE.md vs 자동 메모리

| 구분 | CLAUDE.md | 자동 메모리 |
|------|-----------|------------|
| 작성자 | 사용자 | Claude |
| 포함 내용 | 프로젝트 규칙 및 지침 | Claude가 학습한 패턴 및 선호도 |
| 범위 | 프로젝트, 사용자 또는 조직 | 저장소 단위(작업 트리 전체 공유) |
| 로드 대상 | 모든 세션 | 모든 세션 (최대 200줄 또는 25KB) |
| 사용 목적 | 코딩 표준, 워크플로우, 프로젝트 아키텍처 | 빌드 명령, 디버깅 인사이트, Claude가 발견한 선호도 |

<br>
<br>


# 잘 사용하는 방법

Claude Code 창시자인 **Boris Cherny**는 팀에서 `CLAUDE.md`를 다음과 같이 운영한다고 소개했다.

> "Claude가 실수할 때마다 `CLAUDE.md`에 추가한다. 그러면 다음에는 같은 실수를 하지 않는다."
>
> "수정을 마치면 'CLAUDE.md에 이 규칙을 추가해'라고 말한다."

즉, 반복적으로 발생하는 실수를 규칙으로 축적하는 것이 핵심이다.
<br>
<br>

# 공식 문서 권장 사항

## 넣어야 할 내용

- Claude가 추측할 수 없는 명령어
- 프로젝트의 코드 스타일 규칙
- 저장소 운영 규칙
  - 브랜치 네이밍
  - PR 규칙
- 프로젝트 고유의 아키텍처
- 문서 작성 방식, 톤 등 반복되는 업무 규칙

## 넣지 않는 것이 좋은 내용

- Claude가 코드를 보면 바로 알 수 있는 내용
- 언어의 일반적인 표준 규칙
- 자주 변경되는 정보
- 긴 설명이나 튜토리얼
- 파일별 상세 설명
- "깔끔하게 작성"과 같은 모호한 지시
