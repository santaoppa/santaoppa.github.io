## @RequiredArgsConstructor란?

Lombok 라이브러리에서 제공하는 어노테이션으로, 클래스에 붙이면 "필수 필드"만을 매개변수로 받는 생성자를 자동으로 생성해줌

> 💡 **필수 필드**란?
> - final로 선언된 필드
> - @NonNull이 붙은 필드

``` java
@RequiredArgsConstructor
public class UserService {
    private final UserRepository userRepository;
    private final EmailService emailService;
    private String nickname; //  -> 생성자에 포함 안됨
}
```

위 코드는 Lombok이 컴파일 시점에 아래와 같은 생성자를 자동 생성한 것과 동일하게 동작함

```java
public UserService(UserRepository userRepository, EmailService emailService) {
    this.userRepository = userRepository;
    this.emailService = emailService;
}
```

<br>

## 쓰는 이유?
Spring에서 의존성 주입(DI)을 할 때 생성자 주입 방식을 권장하는데,   
이 때 생성자를 직접 작성하는 코드를 줄여주기 때문에 자주 사용함
- 코드 간결성 : 생성자를 직접 작성할 필요 없음
- 불변성 보장 : final 필드를 사용하므로 필드 값 변경 방지
<br>

``` java
@Service
@RequiredArgsConstructor
public class UserService {

    private final UserRepository userRepository;

    public User findUser(Long id) {
        return userRepository.findById(id).orElseThrow();
    }
}
```
- private final UserRepository userRepository;만 선언해두면
- Lombok이 컴파일 타임에 생성자를 만들어주고
- Spring이 그 생성자를 보고 자동으로 UserRepository 빈을 주입해줌

<br>

## Lombok 생성자 어노테이션 비교

| 어노테이션 | 생성되는 생성자 | 매개변수 |
|------------|----------------|----------|
| `@NoArgsConstructor` | 기본 생성자 | 없음 |
| `@AllArgsConstructor` | 전체 생성자 | 모든 필드 |
| `@RequiredArgsConstructor` | 필수 생성자 | `final` 또는 `@NonNull` 필드만 |

<br>

### 예시 코드
``` java
public class User {
    private final Long id;
    private String name;
    private int age;
}
```

### `@NoArgsConstructor`
``` java
public User() {
}
```
- 매개변수가 하나도 없는 생성자
- 단, final 필드가 있으면 초기화가 안되므로 컴파일 에러 발생
<br>
<br>

### `@AllArgsConstructor`
```java
public User(Long id, String name, int age) {
    this.id = id;
    this.name = name;
    this.age = age;
}
```
- 모든 필드를 매개변수로 받는 생성자
- 필드가 많아지면 생성부 호출부가 길어지고, 어떤 값이 어떤 필드인지 헷갈리기 쉬움
<br>
<br>

### `@RequiredArgsConstructor`
```java
public User(Long id) {
    this.id = id;
}
```
- final인 id만 매개변수로 받음
- name, age는 생성자 밖에서 setter나 다른 방법으로 채워야 함

<br>

### 언제 사용할까?
- `@NoArgsConstructor` : JPA Entity, Jackson 직렬화 등 기본 생성자가 필요할 때
- `@AllArgsConstructor` : 모든 필드를 한 번에 초기화할 때
- `@RequiredArgsConstructor` : `final` 또는 `@NonNull` 필드만 초기화하여 의존성 주입(DI)에 가장 많이 사용
